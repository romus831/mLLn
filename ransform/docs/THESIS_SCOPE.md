# Объём и ограничения проекта MNNL (для пояснительной записки)

## Объект разработки

Реализовано **высокопроизводительное ядро численных операций для обучения ИНС** на C++: тензор `Tensor<float>` с переносом вычислений на GPU (CUDA, cuBLAS, собственные ядра), автоматическое дифференцирование (autograd), модуль **`nn`** с примитивами «слой» (`Linear`, `Sequential`, активации-обёртки).

Поставка: статическая библиотека `tensor`, заголовки, примеры `my_lm`, `xor_demo`, `nn_demo`, опционально `bench_matmul`, unit-тесты.

## Соответствие формулировке темы ВКР

- **C++ и параллелизм:** C++20, CUDA, SIMD/фрагменты OpenMP на CPU-ветках.
- **Библиотека ИНС:** ядро + тонкий слой `nn` + демонстрации; не полный аналог промышленных фреймворков (PyTorch и т.п.).
- **Высокая производительность:** архитектурно ориентировано на GPU и BLAS; количественные сравнения с эталонами — зона описания эксперимента в тексте ВКР (см. `bench_matmul` как источник таблиц).

## Ограничения (честно зафиксировать в работе)

- Тип данных: в основном **float**; нет полноценного шаблонного покрытия double/half.
- Публичный API «нейросети» минимален: нет каталога оптимизаторов, даталоадеров, ONNX и т.д.
- Редукции и часть операций предполагают согласованное состояние CPU/GPU; при расширении кода необходимы дополнительные тесты.
- Сравнение производительности зависит от GPU, драйвера и версии CUDA — в дипломе указывать конфигурацию стенда.

## Направления дальнейшей работы

- Расширение модуля `nn` (дополнительные слои, loss как модуль, простой SGD/Adam как API).
- Регрессионные и нагрузочные тесты, воспроизводимые сценарии бенчмарка.
- Документация API и сценарии установки (`cmake --install`).
